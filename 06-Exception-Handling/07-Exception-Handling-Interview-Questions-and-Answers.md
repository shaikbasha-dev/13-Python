# Basic Exception Handling Questions

## 1. What is an Exception?

### Answer

An Exception is an abnormal event that occurs during program execution and interrupts the normal flow of the program.

---

## 2. What is Exception Handling?

### Answer

Exception Handling is the process of detecting, handling, and recovering from runtime errors without terminating the program unexpectedly.

---

## 3. Why is Exception Handling required?

### Answer

- Prevents abnormal program termination.
- Improves application reliability.
- Provides meaningful error messages.
- Improves user experience.

---

## 4. What is the difference between an Error and an Exception?

| Error | Exception |
|--------|-----------|
| Serious programming issue | Runtime problem |
| Often prevents execution | Can usually be handled |
| Example: SyntaxError | Example: ValueError |

---

## 5. What is the base class for most user-handled exceptions?

### Answer

```python
Exception
```

---

## 6. Name some common built-in exceptions.

### Answer

- ValueError
- TypeError
- ZeroDivisionError
- IndexError
- KeyError
- AttributeError
- FileNotFoundError
- ImportError
- RuntimeError

---

## 7. What is a traceback?

### Answer

A traceback is the detailed error report generated by Python when an exception is not handled. It shows the sequence of function calls that led to the exception.

---

## 8. What is exception propagation?

### Answer

Exception propagation is the process where Python searches up the call stack for a matching `except` block after an exception is raised.

---

# try-except Questions

## 9. What is a try block?

### Answer

A `try` block contains code that may raise an exception.

---

## 10. What is an except block?

### Answer

An `except` block handles exceptions raised inside the corresponding `try` block.

---

## 11. What happens if no exception occurs?

### Answer

The `except` block is skipped, and execution continues normally.

---

## 12. Can one try block have multiple except blocks?

### Answer

Yes.

Python executes only the first matching `except` block.

---

## 13. Can try-except blocks be nested?

### Answer

Yes.

Python allows nested `try-except` blocks.

---

## 14. What is the purpose of `except Exception as e`?

### Answer

It catches most application-level exceptions and stores the exception object in `e` for inspection or logging.

---

## 15. Should `except:` be used?

### Answer

Generally, no.

Catch specific exceptions whenever possible.

---

## 16. Why should try blocks be kept small?

### Answer

Small try blocks make it easier to identify which statement caused the exception.

---

# else and finally Questions

## 17. What is the purpose of the else block?

### Answer

The `else` block executes only if no exception occurs in the `try` block.

---

## 18. What is the purpose of the finally block?

### Answer

The `finally` block always executes and is mainly used for cleanup operations.

---

## 19. Does finally execute after an exception?

### Answer

Yes.

It executes whether or not an exception occurs.

---

## 20. Can try have both else and finally?

### Answer

Yes.

A single `try` statement can contain both.

---

## 21. What is the execution order when no exception occurs?

### Answer

```text
try

↓

else

↓

finally
```

---

## 22. What is the execution order when an exception occurs?

### Answer

```text
try

↓

except

↓

finally
```

---

# raise Keyword Questions

## 23. What is the raise keyword?

### Answer

The `raise` keyword is used to generate an exception manually.

---

## 24. Why do we use raise?

### Answer

To enforce business rules and stop invalid program execution.

---

## 25. Can built-in exceptions be raised manually?

### Answer

Yes.

For example:

- ValueError
- TypeError
- RuntimeError

---

## 26. Can custom messages be provided?

### Answer

Yes.

Meaningful messages can be supplied when raising an exception.

---

## 27. What happens after raise executes?

### Answer

Python creates an exception object and searches for the nearest matching `except` block.

---

# User-Defined Exception Questions

## 28. What is a User-Defined Exception?

### Answer

A custom exception created by the programmer to represent business-specific errors.

---

## 29. Which class should custom exceptions inherit from?

### Answer

```python
Exception
```

---

## 30. Why create custom exceptions?

### Answer

To make business rules clearer and improve readability.

---

## 31. How are custom exceptions raised?

### Answer

Using the `raise` keyword.

---

## 32. How are custom exceptions handled?

### Answer

Using `try-except`, just like built-in exceptions.

---

## 33. Should custom exception names end with "Error"?

### Answer

Yes.

This follows Python naming conventions and improves readability.

---

# Best Practices Questions

## 34. Why should specific exceptions be caught?

### Answer

Because they simplify debugging and prevent unrelated exceptions from being hidden.

---

## 35. Why should exceptions be logged?

### Answer

Logging preserves detailed information for debugging and production support.

---

## 36. Why should `except: pass` be avoided?

### Answer

Because it silently ignores errors, making debugging difficult.

---

## 37. Why should meaningful exception messages be used?

### Answer

They help users and developers understand the actual problem.

---

## 38. Why should input validation be performed?

### Answer

To prevent invalid data from reaching the business logic.

---

## 39. Why is the `with` statement preferred for files?

### Answer

Because it automatically closes the file, even if an exception occurs.

---

## 40. Should exceptions be used for normal program flow?

### Answer

No.

Exceptions should represent exceptional situations, not ordinary control flow.

---

# Scenario-Based Questions

## 41. How would you handle invalid user input?

### Answer

Validate the input, raise or catch appropriate exceptions, and display a meaningful message.

---

## 42. How would you ensure a file is always closed?

### Answer

Use a `with` statement or close the file inside a `finally` block.

---

## 43. When would you create a custom exception?

### Answer

When built-in exceptions do not clearly represent a business rule or application-specific condition.

---

## 44. What is the difference between raise and except?

### Answer

| raise | except |
|--------|---------|
| Generates an exception | Handles an exception |
| Stops normal execution | Recovers from the exception |

---

## 45. What is the difference between else and finally?

### Answer

| else | finally |
|------|----------|
| Executes only when no exception occurs | Always executes |
| Used for success logic | Used for cleanup |

---

## 46. Which block is commonly used to release database connections?

### Answer

The `finally` block (or a context manager where applicable).

---

ZeroDivisionError
```

---

## 48. Which exception is raised for an invalid list index?

IndexError
```

---

## 49. Which exception is raised when a dictionary key is missing?

### Answer

```

---

## 50. Which exception is raised when a file cannot be found?

```

- Catch specific exceptions
- Log important exceptions
- Use `with` for file handling
- Avoid `except: pass`
